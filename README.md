Реализация коллекции данных - бинарное дерево поиска

<h1>Задача</h1>
<p>Реализовать и провести тестовые испытания для коллекции бинарное дерево поиска, содержащей ключи и данные произвольного типа. Тип ключа и тип данных задаются параметрами шаблона класса.</p>
<p>С помощью программы тестирования трудоёмкости исследовать зависимость от размера дерева средней трудоёмкости операций поиска, вставки и удаления. Тестирование провести для двух случаев: для случайного BST-дерева и вырожденного BST-дерева.</p>
Коллекция представляется как упорядоченное, иерархическое, ассоциативное множество элеметнов, между которыми существуют структурные отношения «предки–левые потомки–правые потомки».

<h1>Функционал</h1>
Интерфейс коллекции включает следующие операции:
<ol>
  <li>Конструктор;</li>
  <li>Конструктор копирования;</li>
  <li>Деструктор;</li>
  <li>Опрос размера дерева;</li>
  <li>Очистка дерева;</li>
  <li>Проверка дерева на пустоту;</li>
  <li>Доступ по чтению/записи к данным по ключу;</li>
  <li>Включение данных с заданным ключом;</li>
  <li>Удаление данных с заданным ключом;</li>
  <li>Формирование списка ключей в дереве в порядке обхода узлов по схеме, заданной в варианте задания;</li>
  <li>Дополнительная операция, заданная в варианте задания;</li>
  <li>Запрос прямого итератора, установленного на узел дерева с минимальным ключом begin();</li>
  <li>Запрос обратного итератора, установленного на узел дерева с максимальным ключом rbegin();</li>
  <li>Запрос «неустановленного» прямого итератора end();</li>
  <li>Запрос «неустановленного» обратного итератора rend(). Операции прямого и обратного итератора выполняются по схеме симметричного обхода элементов дерева L → t → R;</li>
  <li>Операция доступа по чтению и записи к данным текущего узла *;</li>
  <li>Операция перехода к следующему (для обратного – к предыдущему) по ключу узлу в дереве ++;</li>
  <li>Операция перехода к предыдущему (для обратного – к следующему) по ключу узлу в дереве --;</li>
  <li>Проверка равенства однотипных итераторов ==;</li>
  <li>Проверка неравенства однотипных итераторов !=.</li>
</ol>
Для тестирования колллекции интерфейс включает дополнительные операции:
<ol>
  <li>Вывод структуры дерева на экран;</li>
  <li>Опрос числа узлов дерева, просмотренных предыдущей операцией.</li>
</ol>

<b>Вариант задания</b>
<ol>
  <li>Алгоритмы операций поиска, вставки и удаления реализуются в итеративной форме;</li>
  <li>Вывод на экран последовательности ключей при обходе узлов дерева по схеме Lt → t → Rt;</li>
  <li>Дополнительная операция: определение длины внешнего пути дерева. Трудоёмкость операции – O (n).</li>
</ol>

<b>Выводы</b>
<p>Характер поведения экспериментальных графиков для вырожденного BST-дерева в целом соответствует прямо-пропорциональной теоретической зависимости.</p>
<p>Характер поведения графиков для случайного BST-дерева экспериментальных зависимостей в целом описывается логарифмической зависимостью, что соответствует теоретическим представлениям о трудоёмкости операций.</p>
<p>Из первого графика видно, что операции обладают приблизительно одинаковой трудоёмкостью.</p>
<p>Из второго графика видно, что самой сложной операцией является операция вставки, а самой простой – операция поиска.</p>
